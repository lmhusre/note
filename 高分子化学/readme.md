# 高分子化学笔记协作说明

这份笔记的主文件是 `高分子化学.tex`。后续不管是人类协作者还是智能体，默认都应把这里当作唯一的正文源文件，不要在 `auxil/`、`out/` 里直接改内容。

## 项目目标

- 把课堂内容整理成结构化、可持续维护的 LaTeX 笔记。
- 优先保证“文字、公式、截图、章节标题”四者一一对应。
- 与课程 PDF/PPT 不一致时，以课程原始资料为准，优先修正文稿和截图，而不是硬保留旧内容。

- `（第六版）第三章 自由基聚合.pdf`

## 推荐工作方式

### 1. 先定位，再改正文

- 先在 `高分子化学.tex` 中定位目标章节。
- 尽量在原有章节结构下补充，不要轻易改乱 `\section` / `\subsection` / `\subsubsection` 层级。
- 如果只是修正截图内容，优先直接替换原图片文件；只有在逻辑结构变化时才同时改 `tex`。

### 2. 图文对应是第一优先级

- 若发现“标题说 A，图片却是 B”，优先修正图片或正文对应关系。
- 公式截图必须只保留公式本体和必要说明，不要把页眉、页码、标题、无关段落一起带进来。
- 对于横向很长的反应式，允许保留必要的参数说明，例如 `n` 的范围、关键试剂或箭头条件。

### 3. 对照课程 PDF 时的原则

- 先检查该知识点是否“完全没有写”。
- 再检查是否“写了主线，但漏了老师 PPT 里常考的小点”。
- 最后检查公式图是否清晰、完整、和文字一致。

## 编译方式

推荐在工作区根目录执行：

```bat
D:\latex0223\run_latex_watch.bat "D:\latex0223\高分子化学\高分子化学.tex" --once
```

说明：

- `run_latex_watch.bat` 会调用 `build_tex_watch.py`。
- 输出 PDF 在 `高分子化学/out/`。
- 日志和中间文件在 `高分子化学/auxil/`。

## 协作约定

- 完成一轮实质修改后再统一编译，不必每改一两行就编译一次。
- 不要手改 `auxil/` 里的 `.aux`、`.toc`、`.log`。
- 不要把临时裁图脚本、接触表、临时渲染页长期留在项目里；如果为了处理截图临时生成了文件，交接前请清理。
- 如果只是重新裁同一张图，优先保留原文件名，减少 `tex` 层的改动。
- 章节补充时，优先补“复习上有价值的知识点”，避免把课件整页原文机械搬运进笔记。

## 交接前检查清单

- 正文是否仍能正常编译。
- `out/高分子化学.pdf` 是否生成成功。
- 新增内容是否和课程 PDF 对应。
- 图片是否清晰、裁切完整、没有无关内容。
- 是否误改了自动生成目录或中间文件。
- 若有未处理的问题，是否在交接说明里点明具体章节和行号。
